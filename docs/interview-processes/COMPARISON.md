## Unique Elements by Company

### Google
| Element | Description | Applies To |
|---------|-------------|------------|
| Googleyness | Cultural fit assessment (purpose, humility, collaboration) | SWE, PM |
| Hiring Committee | Third-party decision makers (removes bias) | SWE, PM |
| Rule of 4 | Minimum 4 interviewers for thorough evaluation | PM |
| Highest PM Technical Bar | PMs expected to work in engineering weeds | PM |
| Team Matching | Post-approval matching with potential teams | SWE, PM |

### Amazon
| Element | Description | Applies To |
|---------|-------------|------------|
| 16 Leadership Principles | Core to every interview round | SWE, PM |
| Bar Raiser | Independent evaluator with veto power | SWE, PM |
| Writing Culture | PR/FAQ documents, writing assessment | PM |
| Data-Driven Answers | Metrics required in all behavioral responses | SWE, PM |
| STAR Method | Essential format for LP questions | SWE, PM |

### Meta
| Element | Description | Applies To |
|---------|-------------|------------|
| AI-Assisted Coding (2025+) | Collaborate with AI in full IDE | SWE |
| Plain Text Coding | No syntax highlighting or autocomplete | SWE |
| "Jedi" Behavioral | Deep collaboration/conflict exploration | SWE |
| 3 Core Categories | Product Sense, Execution, Leadership | PM |
| Follow-Up Interview | Second chance if one round borderline | SWE, PM |

### Apple
| Element | Description | Applies To |
|---------|-------------|------------|
| Non-Standardized | Each team's process is unique | SWE |
| 2 Interviewers Per Room | Simultaneous dual evaluation | SWE |
| Early Termination | Can end interview if below bar by round 4 | SWE |
| Privacy Focus | Security and privacy central to design | SWE |
| Secrecy Culture | Less public process information | SWE |

### Microsoft
| Element | Description | Applies To |
|---------|-------------|------------|
| As-Appropriate (AA) Round | Final round addressing remaining gaps | SWE |
| Being Invited = Good Sign | AA invitation indicates strong performance | SWE |
| 3 Leadership Principles | Create Clarity, Generate Energy, Deliver Success | SWE |
| Team-Dependent Tools | Codility, CoderPad, Excalidraw varies | SWE |

### Netflix
| Element | Description | Applies To |
|---------|-------------|------------|
| Dream Team Philosophy | High-performers only, not "family" | SWE |
| Context Not Control | Autonomy-focused culture | SWE |
| 9 Core Values | Judgment, Courage, Communication, etc. | SWE |
| 50/50 Tech/Behavioral | Equal weight to both categories | SWE |
| Netflix-Style Problems | Real scenarios, not pure LeetCode | SWE |

### Stripe
| Element | Description | Applies To |
|---------|-------------|------------|
| NOT LeetCode Style | Real business scenarios, production code | SWE |
| Bug Bash Round | Debug real issues in provided codebase | SWE |
| Integration Round | Use Stripe API in unfamiliar codebase | SWE |

| Internet Allowed | Full access for docs/syntax in some rounds | SWE |

## Key Takeaways

1. **Amazon is most behavioral-focused** - LP questions in every single round
2. **Google has highest PM technical bar** - PMs must work in engineering details
3. **Meta is piloting AI-assisted interviews** - New paradigm for SWE interviews
4. **Apple is least standardized** - Each team has unique process
5. **Netflix culture fit can be a dealbreaker** - Failing behavioral = rejection
6. **Stripe prioritizes production code quality** - Not traditional LeetCode
7. **Microsoft AA round is a positive sign** - Being invited means you did well
